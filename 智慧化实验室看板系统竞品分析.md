# 智慧化实验室看板系统竞品分析

更新时间：2026-03-18  
研究口径：基于公开官网、产品页、发布页可确认的信息整理，重点关注“智慧化实验室看板 / 驾驶舱 / Dashboard / Analytics”能力。

## 1. 结论先看

市面上的“智慧化实验室看板系统”基本分成两类：

- `国内智慧实验室驾驶舱`：更强调实验室空间、设备、环境、安全、能耗、报警、运维和 3D 可视化。
- `国际 LIMS 运营看板`：更强调样本流转、实验流程、质量合规、角色化工作台、统计分析和钻取能力。

如果我们的目标是做一套既能汇报、又能日常运营的系统，最稳妥的产品形态不是只做一个炫酷大屏，而是：

- `1 个总驾驶舱`：面向领导层，展示全局态势。
- `3 个专业看板`：实验运营、设备环境、质量合规。
- `1 个异常中心`：统一收敛超时、故障、环境超限、库存不足、审核阻塞。
- `1 条钻取链路`：从指标卡片一路下钻到实验室、项目、设备、样本、责任人。

## 4. 公开信息里的可验证设计要点

以下为官网/产品页中可直接确认的设计方向：

- 华大智造 ZLIMS：公开页强调 `丰富工作看板`、`样本全息追溯`、`智能仪器监控`、`实验室预警监测`、`7x24 小时监控大屏` 和 `三维高仿真呈现`。
- 能迪实验室智能管理云平台：公开页明确包含 `大屏展示（3D/2.5D）`、`LIMS 管理系统`、`设备看板`、`健康指数`、`能耗管理`、`综合管控`、`报警管理`、`应急预案`。
- IQR 智能实验室软件：公开页明确写有 `从样品输入到实验结果的全流程自动化管理`，包含 `实验任务下发`、`智能终端管理`、`传感器数据采集与状态展示`。
- 三维天地 SW-LIMS：发布信息强调 `低代码开发平台`、`BI 分析`、`自动排程`、`样本库可视化管理`、`自主定制大屏`。
- Thermo Fisher SampleManager：公开页明确写有 `Pre-configured dashboards`，展示 `resource availability`、`stock information`、`location status`、`lab performance`，并支持交互式钻取。
- LabWare：公开页明确写有 `Configurable user dashboards`，支持按 `laboratory managers`、`analysts`、`quality control personnel` 等角色个性化展示。
- 1LIMS：首页截图直接展示 `订单状态计数`、`月度订单柱状图` 和 `最新订单列表`。
- LabVantage：公开页强调 `benchtop to boardroom`，明确面向实验端与管理层双重场景。
- STARLIMS Advanced Analytics 2.0：公开页明确强调 `responsive and configurable dashboards`、`dynamic reporting`、`natural language processing`、`drill-down analytics`。
- eLabNext：帮助中心公开文档可以确认其移动端支持 `扫码查看样本`、`设备预约`、`库存浏览`、`低库存通知`。

## 5. 行业内最常见的页面结构

### 5.1 首页总驾驶舱

常见布局通常是：

- 顶部：实验室总数、设备在线率、当日样本量、任务完成率、异常数、库存预警数。
- 左侧：样本流程分布、任务进度、实验室分布、项目状态。
- 中部：实验室地图、楼层平面、3D 场景、关键设备状态。
- 右侧：实时告警、待审核、逾期任务、环境超限、设备故障。
- 底部：趋势图与明细表，如近 7/30 天 TAT、异常趋势、最近告警、最近任务。

### 5.2 实验运营看板

- 样本接收量、在检量、待审核量、已出报告量。
- 各实验阶段漏斗，如接样、建库、上机、分析、复核、报告。
- TAT 趋势、超时分布、项目产能、人员工时。
- 样本/任务/项目明细表，支持筛选、导出、钻取。

### 5.3 设备环境看板

- 设备在线率、稼动率、维修率、校准到期数。
- 温湿度、压差、洁净度、气体、门禁、视频、能耗。
- 告警级别、告警处理时长、重复故障排名。
- 房间、楼层、实验区视角切换。

### 5.4 质量合规看板

- OOS/OOT、CAPA、偏差、变更、审核追踪。
- 电子签名进度、批次放行状态、审计追踪完整性。
- 方法执行一致性、复核积压、报告合规率。

## 6. 竞品共性规律

从这些产品里能看出几个很稳定的设计规律：

- 几乎所有产品首页都会先放 `状态型 KPI 卡片`，用户先看总数和异常，再看图表。
- 真正高频使用的不是“炫酷大屏”，而是 `KPI + 队列 + 明细表` 的工作台结构。
- 国内产品更容易把 `设备、环境、安全、能耗` 放到同一个驾驶舱。
- 国际产品更强调 `角色化视图`，实验员、主管、QA、管理者看到的首页并不一样。
- 成熟产品几乎都会提供 `钻取能力`，不会停留在一级大屏。
- 只做展示、不能联动业务动作的看板，长期使用率通常会下降。

## 7. 对我们项目的产品建议

### 7.1 建议采用的产品结构

建议做成 `1 + 3 + 1`：

- `总驾驶舱`
- `实验运营看板`
- `设备环境看板`
- `质量合规看板`
- `异常与待办中心`

### 7.2 建议首页信息架构

首页可以按以下顺序组织：

1. 第一屏先给领导看结果：样本总量、任务完成率、TAT、设备在线率、异常数、库存预警。
2. 第二屏给主管看过程：样本阶段分布、实验流程漏斗、实验室产能排名、人员负荷。
3. 第三屏给运维和 QA 看风险：环境超限、设备故障、审核阻塞、CAPA、校准到期。
4. 所有图表都支持点进去看明细：实验室、项目、设备、样本、责任人。

### 7.3 建议重点抄的能力

- 抄 `ZLIMS` 的样本全链路与实验室态势表达。
- 抄 `能迪` 的设备环境、报警、能耗一体化驾驶舱。
- 抄 `SampleManager` 的 KPI 卡片 + 交互式钻取分析。
- 抄 `LabWare` 的角色化 dashboard 逻辑。
- 抄 `1LIMS` 的简洁首页骨架，避免信息过载。
- 抄 `STARLIMS` 的趋势分析和下钻能力。
- 抄 `eLabNext` 的移动端扫码、库存提醒、轻交互。

### 7.4 不建议直接照搬的方向

- 不建议把首页做成纯 3D 展示，业务细节会被遮蔽。
- 不建议把所有模块都堆在一个大屏里，领导和一线用户诉求不同。
- 不建议只有汇总图，没有明细表和责任人维度。
- 不建议只做视觉效果，不做异常闭环和业务跳转。

## 8. 你们项目的差异化机会

如果我们不是做标准 LIMS，而是做“报告系统看板”，差异化可以放在这几个点：

- `报告链路透明化`：接样、检测、复核、签发、发送、回执全链路可追踪。
- `报告质量风险前置`：把退回、补样、复测、签发阻塞、超时风险提前显性化。
- `业务与设施同屏`：既看报告与样本，也看设备、环境、异常，形成真正的智慧实验室驾驶舱。
- `多角色切换`：领导、平台主管、实验主管、QA、设备管理员看到不同首页。
- `异常闭环`：每个红点都能点进去看到原因、责任人、处理时限和当前状态。

## 9. 推荐纳入首版的指标清单

建议首版至少覆盖以下指标：

- 样本类：接样数、在检数、逾期数、补样数、复测数、已出报告数。
- 报告类：待复核、待签发、已签发、超时报告、退回报告、报告准确率。
- 流程类：各节点平均耗时、TAT 达成率、节点阻塞数、任务完成率。
- 设备类：在线率、故障数、维修中数量、校准到期数、稼动率。
- 环境类：温湿度超限次数、压差异常次数、报警处理时长。
- 质量类：偏差数、CAPA 数、OOS/OOT 数、审计追踪异常数。
- 资源类：试剂库存预警数、耗材可用天数、关键物料缺口。
